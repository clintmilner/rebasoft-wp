Installation and Configuration

Application Auditor is provisioned using a simple installer that can be run on Windows Server operating systems. The installer can be used for initial installation, software version upgrades and changes in system parameters.

Hardware Requirements

The type of system required to run Auditor depends on the number of source devices collected from and length of data storage. The following requirements are a guideline; the only way to determine requirements is by testing the software’s performance in a specific network environment. To this end Rebasoft has packaged a 7-day trial download.

CPU

• Minimum CPU should be a dual core 2.4GHz processor.

• For every additional 50 data sources, add an additional core/CPU

Memory

• Allow for base system of 2GB RAM and 100MB for each data source sending data

Disk

Each IPFix Record require 120Bytes of disk space for every 1 Million of stored flow records require allow 120MB of disk space

• SATA II (1.5GB/s+) disks are preferred on small to medium systems

• SAS (3GB/s+) for larger systems (more than 100 devices)

Software Requirements

• Windows 2003+ English Server

• Application Auditor should be installed as a local administrator

• For systems requiring more than 4GB of RAM a 64bit OS (JRE) is required

Installing Application Auditor

Installation is achieved by executing the Application Auditor installation wizard. This wizard guides administrators through the installation or updating process and allows the specification of a number of parameters – it should only take a few minutes to install the system.

The installer can be run whether performing a new installation or updating an existing system. In the case of an upgrade, data is preserved; Rebasoft does, however, advise administrators to perform a data back-up regularly using pgAdmin (see appendix) and especially prior to system upgrade.

Note

Due to windows security, we recommend that the installer be run as the Local Administrator
