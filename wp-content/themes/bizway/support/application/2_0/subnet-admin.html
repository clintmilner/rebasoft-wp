Subnet Settings

Subnet administration allows you to specify which network segments you wish to monitor and report upon within Application Auditor. If subnets are overlapping, this can be addressed by the specification of devices and interfaces on which the subnet data has been collected. In addition to filtering views of both monitored and un-monitored subnets, reporting displays device and interface statistics where an available subnet has been seen on a device which is currently being monitored.

Subnets are defined and managed in the 'Subnets' section from the Admin selector:

To add a new monitored subnet, enter the credentials required and select the device and interface on which you wish to monitor the subnet. Once the details have been added, select the ‘Add New Subnet’ button:



note: Application Auditor uses device + interface + subnet to uniquely identify subnets. This means that overlapping subnets on a single instance of application auditor is supported.
