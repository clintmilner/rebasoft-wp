Syslog processors

Application Auditor can receive application usage data from source devices exporting log messages via the syslog protocol. Supported source devices fall into two categories - proxy servers and firewall/access control devices.

Syslog Listeners are configured via Receivers administration. Application Auditor V2.0 provides the ability to accommodate variations in proxy server messages via the Syslog processors administration dialog:

To access Syslog processors, select "Syslog processors" from the administration drop down in the top right hand of Application Auditor:

Current syslog processing is shown next to the receiver:

Any messages received, and unprocessed, are shown to administrators. These messages can be analysed and added to the Application Auditor database using a regex as a powerful way of manipulating incoming data.

To add an unprocessed message to Application Auditor, simply select Add pattern match:

It is then simply an exercise in applying the regex to the message and testing

The "test pattern" button can be used to verify the pattern match before adding to the database.
