Browsing sites and devices

Rebasoft Auditor allows users to logically drill down from Sites to Devices to VLANs, Interfaces or MACs within a few clicks. Navigation is simply an exercise in expanding or collapsing sites that you have defined, clicking hyperlinks or through the Site visuals. An expanded view of all sites/devices can be seen using the collapse/expand all controls at the top of the list:

Filtering the list of sites & devices

To help with getting rapidly to the right device use the filter. To access the Sites & Devices filter, click the icon next to the heading:

To find the device of interest, simply place the cursor in the box and start keying the device name, Rebasoft Auditor will dynamically Filter the list.

Device names are colour coded:

  • White = OK
  • Red = SNMP not OK or has failed the latest poll
  • Blue = Device hasn't been polled yet (start-up or after a rescan)
  • Grey = Device is unmanaged

A site summary is displayed above the map including any White list (if set up) alerts that have been seen. Note: the figures shown are the maximum no. seen concurrently. It is also possible for users to position sites and devices on the visuals and save them for the next user using the "save" icon next to the Site visual.

Within the Site overview, a series of time based summary reports for the site are display:

  • MAC Addresses seen
  • Top 5 Devices over time
  • IP Addresses seen
  • Devices seen in use
  • Interfaces seen in use
  • Interfaces by MAC count
  • Interfaces by IP count

It is also possible to report on “Down interface”, (this can be used as the basis to find available ports), MAC’s by Vendor, MAC’s by IP count, MAC’s by Interface count and IP’s by MAC count.

Rebasoft Auditor will, during initial device discovery, poll the interface clock for last used status and use this as a start point for last used information. Where interface last used status is unavailable, Rebasoft Auditor will use the polling information. Subsequent polling for interface status will update Rebasoft Auditor down interface reports.

From the Sites and devices menu, selecting a device will show a device overview, key reports, interface statistics and lists of the interfaces and VLANs on the device:

“Device Details” provides device level information; showing the hostname, IP address/DNS name (Hostname), OiD type (this can be customised see Vendor OID ) and a description.  It is also possible to add user defined Tags, these can be added manually or imported from static databases (eg audit data giving location, user, owner etc.) to give other information which would be useful when following up an issue. All information in the device details area is searchable within the database. For each device, below the device details, users can view statistics for the device.

Please note the “Tools” line – this is where Integration for devices appears. We have shown a java SSH/Telnet in the image as an example

Below the device details are a series of reports and a list of the active (with MAC addresses seen) and inactive (without any MAC addresses seen) interfaces for the device. In the same way as with Site overview, the reports can be zoomed and selected for additional detailed reporting
