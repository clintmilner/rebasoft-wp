Installing Auditor

Rebasoft Auditor is supported on Windows and Linux (see System requirements for more details)

System parameters

Rebasoft Auditor relies on SNMP polling device CAM tables as part of information collection. The software has been extensively tested in large networks and against different device types and vendors. System parameters are set as a result of the testing and in line with system recommendations below.

Experience shows that it takes around 1 minute for a device to respond to poll of the CAM table. The system default is for the polling cycle to be completed every 30 minutes. The objective therefore is to set the number of concurrent poller threads to be sufficient to complete to information collection and processing within the allotted time.

We recommend that 1 thread be allowed for each 10 devices to be polled. The installation default is for 20 threads, this would allow support of up to 200 devices (with between 5000 and 8000 interfaces). The thread count can be increased subject to network and server load considerations.

Installing Rebasoft Auditor on Windows

Auditor is provisioned using a simple installer that can be run on Windows Server operating systems. This installer can be used for initial installation, software version upgrades and changes in system parameters. The installation wizard guides administrators through the installation or updating process and allows the specification of a number of parameters – it should only take a few minutes to install the system.


Note:

Due to windows security, we recommend that the installer be run as the Local Administrator

To start installation, simply execute the installer:


  1. Click next…

  2. Click “I Agree” to accept the licence agreement …

  3. Select components to install and click next …

  4. Enter desired http/https port together Threads and RAM settings desired (the RAM setting dictates the amount of the RAM allocated to Auditor to run, if you are polling a large number you will need to increase  this value)

    The number of threads is a setting for enabling parallel SNMP polling of your network this value should be increased if the devices increase the number of parallel pollers Auditor should use. The larger the number of devices polled, the more threads should be set (as a guide 1 thread per 20 devices polled, up to a maximum of 50). Larger numbers of threads requires more RAM.

    If it becomes necessary to amend the number of threads or the RAM simply rerun the installer and change them.

    If the http or https ports entered are already in use by another application, the installation will prompt you to select an alternative port

  5. Specify installation directory for installation and click next …

    Note:

    The Java installer does not permit spaces in the specification of the destination folder

  6. If Auditor is already installed, the installer will now stop the MACAuditor service in preparation for the upgrade of files

  7. If not installed Rebasoft Auditor will install PostgreSQL and JRE

    Sometimes, due to windows permissions, postgres will not install. The installer will terminate and a message will request that postgreSQL be installed manually.

  8. Rebasoft Auditor installs as a system service, a command window should appear during this process

  9. Once Rebasoft Auditor installation is complete (administrators can view the details of where files have been installed by clicking the details button) click next…

Installing Rebasoft Auditor on Linux

Auditor requires PostgreSQL V9.x or later and Java (JRE) 6 or above. These should be installed in the normal way using the package administration utility supplied with the Linux distribution. Auditor has been tested on SUSE and Red Hat Linux distributions.

The following instructions assume installation on a SUSE Linux system.

Note:

P0stgr3s_2ME should be used for the default PostgreSQL password
  1. Install libpq5

    Download libpq5-9.0.4-54.1.x86_64.rpm

    rpm -i libpq5-9.0.4-54.1.x86_64.rpm
  2. Install libecpg6

    Download libecpg6-9.0.4-54.1.x86_64.rpm

    rpm -i libecpg6-9.0.4-54.1.x86_64.rpm
  3. Install PostgreSQL server

    Download postgresql-server-9.0.4-54.2.x86_64.rpm

    rpm -i postgresql-server-9.0.4-54.2.x86_64.rpm
  4. Install PostgreSQL

    Download postgresql-9.0.4-54.2.x86_64.rpm

    rpm -i postgresql-9.0.4-54.2.x86_64.rpm
    /etc/init.d/postgresql start
    su - postgres
    psql
    create user rebasoft PASSWORD 'rebasoft';
    create database rebasoft owner = rebasoft;
    \c rebasoft
    create language 'plpgsql';

    To exit the terminal, type \q

  5. Install JRE

    Download from oracle.com

    chmod 755 jre-6u25-linux-x64-rpm.bin
    ./jre-6u25-linux-x64-rpm.bin
    ln -s /usr/java/jre1.6.0_25/ /opt/java
  6. Install Auditor tar -vxf rebasoft_auditor_unix_1078.tgz -C /
    cp /opt/Rebasoft/MACAuditor/resources/unix/start.template /opt/Rebasoft/MACAuditor/resources/start
    cp /opt/Rebasoft/MACAuditor/resources/unix/macauditord /etc/init.d/macauditord
    chkconfig  --add macauditord
    /etc/init.d/macauditord start

If the server fails to start due to ident issues, edit the pg_hba.conf file: /var/lib/pgsql/data/pg_hba.conf
Append "host rebasoft 127.0.0.1/32 md5

and restart postgresql /etc/init.d/postgresql restart
/etc/init.d/macauditor restart

Log into Auditor, set logging to INFO from the system settings page
