Capacity view

Auditor, as it collects data from your network over time, enables administrators to track interface usage over time. The reporting can be done at system, site and device level; capacity Reporting will adjust the output relevant to your area of interest accommodating networks large and small.

Auditor not only tracks interface status, but also records MAC, IP and users attached to the interface. Thus a real-time, up-to-date view on real interface availability can be shown.

Capacity view, therefore, provides two key functions:

  1. The ability to report, over time, on asset utilization across the network
  2. generate change plans from real-time, dynamic data; the change plans can be compared against actual connections showing any variation from the plan.

The reports generated by Auditor can be exported to pdf, csv, xml and more.

Using the Rebasoft Auditor Capacity Module

Auditor Capacity view is accessible via the Capacity option from the view selector in the header section of the Web Interface.

Once selected, you are presented with a System overview comprising information from devices monitored by Auditor. This content is dynamic and will change based upon your selected view using the Sites & Devices Navigator.

By default, the Capacity view will report on:

  • Current interface count by status
  • Historical interface count by status
  • Current interface count by type and status
  • Least used devices by interface usage
  • Top 10 underutilised sites by interface usage
  • Devices in use (trended count)
  • Interfaces In Use (Trended count)

Capacity Reporting Menu Bar

Users are able to specify criteria such as output format, report start date and duration of reports for visibility of their required elements.  Output options available areas follows:

  • HTML
  • PDF
  • XML
  • JSON
  • CSV

Clicking on the 'Date' window will produce a date picker, allowing you to specify the start date and reporting period for your report.

Note:

Reporting Filters are assigned to the current view that is selected (such as site or device). If at any time you wish to return to the top level for Capacity view, select the  icon next to the Capacity option from the view selector in the Header section of the Web Interface.

In addition to the above, the standard navigation buttons are also available throughout the Capacity view.

Change Plans

Capacity view allows users to generate and adjust Change plans. Once finalised Change plans can be compared with actual connections, allowing users to see where there are variances in the plan and 'close' a plan once completed.

List your Change Plans by selecting a view type on the left navigator ('By User' or 'By Status').

Creating a Change Plan

To create a Change Plan, enter the name of your plan, add any notes required and click the 'Add' button:

Once created, your plan will appear in the Change Plan list as a new plan:

To add Interfaces an existing 'open' plan, select an interface that you would like to add to the plan and click the 'Add To Plan' button associated with the current listing:



Note:

Interfaces can also be added to a change plan from within the operations view at the Interface overview screen. Simply click the add to capacity plan icon:

Specify the required plan to which you wish to apply the change and choose whether the Interface is a 'Source' or 'Destination' Interface.

Once you have added the Interface to your plan can save the changes by clicking 'Save Plan'. This will update with the current changes.

Plans will automatically be set to the following status:

  1. New
  2. Source Set or Destination set
  3. Source and destination set

To view or amend the plan, click on the plan name to view the criteria specified for that plan. Interface "To" and "From" mappings can be changed by 'dragging' the ID label up or down the change list – the cursor changes to a cross.

When happy that your Change Plan is complete, select Finalise to lock the content of the Change Plan.

Note:

Once a change plan has been finalised you CANNOT edit the content of that plan

At any point after destination interfaces have been set, 'Show Current' will display the current details of the "To" interface. If there is a current device this will be shown

"Show current" will allow users to see whether an interface is free prior to a change, or check whether the "from" and "to" MAC addresses match following a change.
