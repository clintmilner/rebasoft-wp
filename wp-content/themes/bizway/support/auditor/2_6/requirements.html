System Requirements

The type of system required to run Rebasoft Auditor depends on the number of devices to be polled for management reporting. The following requirements are a guideline; the only way to determine requirements is by testing the software’s performance in a specific network environment. To this end Rebasoft has packaged a 7-day trial download.

Hardware requirements

CPU:

  • Minimum CPU should be a dual core 2.4GHz processor.
  • For every additional 200 devices add an additional core/CPU

RAM:

  • Base system of 2GB RAM
  • For each device (Switch/Router) allow 2MB of RAM
  • For a blade based Switch (i.e. Cisco 6500) allow 2MB of RAM for each blade

Disk:

  • Disk - 10MB of disk space per device per month
  • SATA II (1.5GB/s+) drives for small to medium systems (up to 500 network devices polled)
  • SAS (3GB/s+) drives recommended for larger systems (greater than 500 devices)

Software Requirements

  • Windows 2003/2008 server or later
  • Linux (SUSE, Red Hat, Centos, Debian, Ubuntu tested)
  • Solaris

VMware is a supported environment

Internet Explorer V7 or later, Mozilla V3 or later and Chrome V11 or later (JavaScript must enabled in the browser).

1200 x 1024 pixel minimum screen resolution is recommended for client devices accessing Rebasoft Auditor.

Java Runtime (JRE) V1.6 and PostgreSQL are required for the application to function. If not installed, these will be installed by the installer on Windows platforms. Linux and Solaris may require these components to be installed separately.

Note: For systems requiring more than 3GB of RAM, 64bit JRE is required
