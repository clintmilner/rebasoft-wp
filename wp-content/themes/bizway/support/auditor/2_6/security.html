Auditor for Risk Management

Auditor provides a security solution for organisations by supporting IT risk management. Auditor V2.6 does this in three main ways:

  1. Ensuring that organisations are aware of the extent of the infrastructure. This is the foundation of any risk management exercise, as it defines the risk domain. If the domain does not contain the whole infrastructure, then any risk management exercise may have security gaps

     

  2. Classification and approval of all connected devices. Starting with discovered MAC addresses, Auditor allows security administrators to classify and approve devices. This enables organisations to ensure all devices can have appropriate security controls applied

     

  3. Alerting. Auditor provides three main security alert capabilities that enable organisations to respond to potential threats in real time.
    • Locate and tracking potential rogue devices
    • Locate and alert on new / unknown devices
    • Locate and alert new infrastructure uplinks - enabling organisations to spot potential rogue access points/ hubs as they try to bypass network security controls


Network Edge

Understanding the boundary or edges of the infrastructure is a key part in defining the risk domain and, thus, being sure that all devices connected to the infrastructure can be seen. Network edges exist in many forms, but all essentially provide access to or from different parts of the network. Network edges can be either wired or wireless.

Rebasoft helps with Risk Management by locating and classifying network edges. A simple workflow allows administrators to identify and approve all edges. By classifying and approving all edges, administrators can ensure that all major ingress and egress points are under control, and that no unauthorised network infrastructure device or uplink can be used for network access.

Network Edge Interface reports, therefore, starts by focussing on differentiating network device interconnections (edges) from network interfaces used for end-device connections (access ports).

Note: Administrators define an edge through the "Uplink Threshold MAC Count" setting in System Settings. The default setting is 3, thus any interface with more than 3 MAC addresses connected is classified as an uplink or network edge

 

Once a list of edge interfaces has been identified the system classifies them by type. Rebasoft, by default, uses the IANA interface definitions (see http://www.iana.org/assignments/ianaiftype-mib for more details) These definitions can be amended in system settings to suit an organisations reporting requirements. Security administrators can then correlate this list and approve. Each uplink can be investigated to see the connected devices. Auditor shows the connected devices in the navigator to help see where the most devices are connected and thus provide a cue as to where the largest risk edges are.

Network Edge is a real-time process that is constantly updated:

  1. Auditor profiles all edge interfaces based on the MAC threshold settings
  2. Alerts are generated each time a new network uplink is discovered - this alert may be optionally syslogged to a SIEM or other centralised alert/log management system
  3. Alerts are also generated each time a network uplink is classified or approved for audit purposes

    End device coverage

    Once the extent of the risk domain has been defined, Auditor will be seeing all the connected devices. At the lowest level, a device must have a MAC address in order to communicate. Auditor tracks MAC addresses in real-time.

    By default MAC addresses are separated into Infrastructure and non-infrastructure categories. Auditor knows which MAC addresses are part of the devices being managed by it, thus these are automatically classified as type Infrastructure.

    Auditor also categorises devices by Vendor (MAC Vendor) in the absence of any other data. MAC Addresses seen on the network will also show IP address, user or host name if available:
