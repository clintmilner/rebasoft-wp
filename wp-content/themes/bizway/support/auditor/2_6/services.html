Services administration

Auditor provides organisations with the ability to configure services for various purposes:

Radius Accounting | SNMP trap receiver | DHCP listener

Note: With all the services configured below, please ensure that the relevant Windows Firewall or iptables settings allow inbound traffic on the specified listener ports.

Radius Accounting

Radius accounting provides the ability to use Radius as a mechanism to collect userID information for use in user forensics

Configure Radius forensics by:

  1. Configure the listener service - this just needs a name and a port number which Auditor is required to listen for radius traffic

  2. Start the listener service

  3. Configure host and secret for the relevant remote hosts

SNMP Trap receiver

SNMP Trap receiver provides the ability to receive and process MAC notification messages from network enabled devices. This enables Auditor to process device connections in real-time. MAC notification implementation varies from vendor to vendor, but provides the most reliable mechanism for real-time device connections.

Configure SNMP trap listener by:

  1. Configure the listener service - this just needs a name and a port number which Auditor is required to listen for SNMP traps

DHCP listener

DHCP provides the ability to glean information about end-devices as they connected to the network when they request an IP address from a DHCP server. The DHCPDISCOVER message provides a rich source of information about the requesting device (in order for it to obtain the best service from the DHCP server). In addition to the end device details, DHCP can provide a degree of real-time notification of devices connecting to the network (this may be useful if MAC notification is unavailable)

Note: Unlike MAC notification, DHCP cannot provide a switch port location for the end device. Auditor will only localise the device during the normally SNMP polling cycle

Auditor Core provides the ability to "tap in" to the DHCP messages via a configurable DHCP listener service. The information in the DHCPDISCOVER packet is cross-referenced against known message types and a match is tagged to the MAC element.

Configure DHCP listener by:

  1. Configure the listener service - this just needs a name and a port number which Auditor is required to listen for DHCP, and a probability match setting. The probability match is used in ensuring a full fingerprint match in order to generate tags for MAC forensics.

  2. Start the listener service

All the listener services show statistics for packets received and processed

Note: The DHCP messages will need network administrators to configure the Auditor Core server as a destination address for IP helper/BOOTP broadcast messages. Please consult your vendor configuration guide for details. IP helper will then replicate the DHCPDISCOVER broadcast messages to the Auditor server for processing.
